QuestionMetric / EvidenceWhy It Is UsedImportant Caveat
Do changed pixels receive higher scores?AUROCThreshold-free ranking check from binary classification.Can look good even if thresholded maps are poor.
Does the method work under class imbalance?Average precision / PR-AUCChange pixels are rare, so precision-recall behavior matters.Still threshold-free; does not itself produce a binary mask.
Can the score become a mask without labels?Otsu F1 and IoUOtsu gives an unsupervised histogram threshold.Otsu can fail if score histograms are not bimodal.
What is the optimistic upper bound?Best F1 / best IoUShows whether the score contains useful signal at any threshold.Uses labels to choose threshold; not deployable.
Is DS just raw difference again?Correlation with raw L2Detects redundancy with naive spectral-difference magnitude.Low correlation is not automatically good; it must still match labels/maps.
Does spatial support help?Maps + metrics across global/window/patchDirectly tests Sensei's spatial-information concern.Patch/window choices are adaptations, not original DS theory.

Source anchors: OSCD / FC-Siamese work reports change-class precision, recall, and F1; scikit-learn defines AUROC and average precision; Otsu thresholding is the standard histogram-based unsupervised threshold check.
