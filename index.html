Pipeline From Data To Claim

This is the chain you should be able to explain in seminar: sample definition, subspace construction, scoring, map generation, metrics, then claim.

Current OSCD Spatial DS Flow

OSCD city folder
  -> load 13 Sentinel-2 tif bands for pre and post
  -> normalize bands using OSCD band stats
  -> choose sample definition:
       global pixel: 13 x N
       patch3: 117 x N_patch
       patch5: 325 x N_patch
       window128: many 13 x N_window matrices
  -> fit rank-r PCA subspace for pre and post
  -> build canonical DS basis from the two PCA bases
  -> score ||D^T(sample_post - sample_pre)||^2
  -> reshape scores into a change map
  -> compare to OSCD label with AUROC, AP, F1, IoU, Otsu
  -> inspect map and decide next experiment

What "Meaningful" Means Here

For the current experiment, meaningful is not a vague word. It means the score map survives baseline pressure, spatial support actually changes behavior, and the map aligns with labels without becoming raw L2 again.
